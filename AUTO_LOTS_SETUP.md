# Auto Lots Setup

Этот файл нужен, чтобы завтра быстро подключить новые панорамы к участкам из KML.

## Как связаны данные

- Геометрия участков лежит в `data/lots.geojson`.
- Пользовательская таблица берется из Google Sheets:
  `1mdAVnkyEWr9FRAd0f_JRygSIuxwFdzOWXnuqEkJERwg`, лист `2123234659`.
- Связь идет по кадастровому номеру:
  - в GeoJSON: `lot_id` / `kad_nomer`
  - в пользовательской таблице: `Кадастровый номер`

Если в пользовательской таблице статус пустой, участок на панораме не показывается.

## Статусы

- `Свободно` - зеленый участок
- `Бронь` - желтый участок
- `Продан` или `Продано` - красный участок

Остальные данные, например цена, площадь и адрес, берутся из пользовательской таблицы и показываются в окне участка.

## Колонки для панорам

В главной таблице тура для каждой панорамы, где должны появляться участки, нужны такие колонки:

| Колонка | Что писать |
| --- | --- |
| `scene_lat` | широта точки съемки панорамы |
| `scene_lng` | долгота точки съемки панорамы |
| `scene_altitude_m` | высота съемки в метрах |
| `heading_offset` | поворот участков по горизонту |
| `pitch_offset` | сдвиг участков вверх/вниз |
| `lot_radius_m` | радиус показа участков в метрах |
| `lot_display_mode` | режим показа участков |
| `lot_group_size` | примерный размер одной группы |

Обязательные колонки: `scene_lat`, `scene_lng`, `scene_altitude_m`.

Если `lot_radius_m` пустой, сайт использует `1400` метров.
Если `heading_offset` или `pitch_offset` пустые, сайт использует `0`.

## Режим групп

Чтобы на панораме сначала показывались не все участки, а группы, в строке этой панорамы поставьте:

| Колонка | Значение |
| --- | --- |
| `lot_display_mode` | `groups` |
| `lot_group_size` | например `20` |

Если `lot_display_mode` пустой, панорама работает по старой логике и показывает участки сразу.

В режиме `groups` сайт:

1. Берет участки вокруг этой панорамы.
2. Делит их на группы примерно по `lot_group_size`.
3. Сначала показывает карточки групп со статистикой.
4. После нажатия на группу показывает только участки внутри этой группы.

## Как настроить завтра

1. Загрузить новую панораму в Cloudflare R2.
2. Добавить новую сцену в главную таблицу тура.
3. Заполнить `image_url`, `scene_id`, название сцены и координаты съемки.
4. Открыть тур и подобрать:
   - `heading_offset`, если участки повернуты левее/правее нужного места;
   - `pitch_offset`, если участки слишком высоко или низко;
   - `lot_radius_m`, если надо показать больше или меньше участков.
5. Если на этой панораме нужно показывать участки группами, поставить `lot_display_mode = groups`.

Подбор обычно делается так: сначала выставить координаты и высоту, потом крутить `heading_offset`, потом чуть поправить `pitch_offset`.

## Важно

Файл `data/lots.geojson` загружается через `fetch`, поэтому локально его лучше проверять через маленький сервер, а не просто открывать HTML как файл.
